Three ideas to reduce the Task → Mapping → Endpoint round-trip

Concept 1
Task as Pipeline Hub
The task detail page shows the full pipeline — source endpoint, mapping, target endpoint — as linked cards. Edit any layer without leaving the task.

Concept 2
Slide-over Editor
Clicking "Edit mapping" or "Edit endpoint" opens a panel on the right — you stay on the task page. Save closes the panel, ready to re-run immediately.

Concept 3
Error → Direct Field Link
Run log errors include a link that opens the mapping editor pre-scrolled to the exact failing field. No searching required.

Concept 4
Breadcrumb Drill-down
Task → Mapping → Endpoint as one continuous drill-down. The browser back button works naturally, and context is never lost.
